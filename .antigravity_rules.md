# Wook's AI and Marketing Blog: CMS Rules

이 문서는 사용자가 Antigravity(AI)에게 **웹 문서 요약**이나 **블로그 포스트 작성**을 지시할 때, AI가 반드시 준수해야 하는 편집 및 깃허브 커밋 규칙입니다.

## 1. 포스팅 목적 및 톤앤매너
* **카테고리**: `Marketing`, `AI News`, `AI Learnings`, `Career` 중 하나를 선택합니다.
* **톤앤매너**: 마케팅과 AI에 관심 있는 주니어~시니어 실무자들이 읽기 좋은, 통찰력 있고 전문적이면서도 너무 딱딱하지 않은 '비즈니스 캐주얼' 어투(해요체/하십시오체 혼용)를 사용합니다.

## 2. 파일 생성 및 저장 규칙 (File System)
사용자가 "발행해" 또는 "포스팅해" 라고 승인하면, AI는 **반드시** 로컬 파일 시스템에 마크다운 파일을 생성해야 합니다.
* **경로**: `/Users/wook/WookAi/Booklog/content/posts/`
* **파일명 룰**: `YYYY-MM-DD-english-title-slug.md` (예: `2026-04-23-tesla-ai-analysis.md`)

## 3. 프론트매터 (Frontmatter) 필수 포맷
파일 최상단에는 반드시 아래와 같이 YAML 프론트매터를 포함해야 프론트엔드에서 정상적으로 읽을 수 있습니다.
```yaml
---
title: "포스팅의 매력적인 제목"
date: "YYYY-MM-DD"
excerpt: "목록(List) 페이지에서 보여질 1~2줄짜리 강력한 흥미 유발 요약"
category: "위 4가지 카테고리 중 택 1"
---
```

## 4. Git 커밋 및 자동화 (Publishing)
파일을 생성한 직후, AI는 터미널을 이용해 GitHub로 변경사항을 반드시 Push 해야 합니다.
* `git add content/posts/[생성된파일명.md]`
* `git commit -m "Auto-publish: [포스팅 제목]"`
* `git push`

(이 과정을 통해 Cloudflare Pages나 Vercel의 CI/CD가 알아서 자동 배포를 수행합니다.)

## 5. 논문 검색 및 리서치 규칙 (Academic Research Rules)
앞으로 이 프로젝트(PC)에서 논문을 검색하고 수집할 때는 반드시 아래의 가이드라인을 따릅니다:
1. **지정된 검색 엔진 사용**: 논문 검색 시 목적에 따라 반드시 다음 사이트들을 우선적으로 활용합니다.
   * **[OpenAlex](https://openalex.org/) / [Google Scholar](https://scholar.google.com/)**: 방대한 학술 자료의 기본 검색 및 원문 탐색용
   * **[Semantic Scholar](https://www.semanticscholar.org/)**: AI 기반 검색 및 특정 이론의 발전 방향(인용 그래프) 추적용
   * **[Consensus](https://consensus.app/)**: 질문형 검색을 통해 동료 평가(Peer-reviewed)를 거친 논문들 기반의 '과학적 합의점' 도출 및 팩트 체크용
   * **[Elicit](https://elicit.com/)**: 여러 논문에서 실험군 크기, 측정 지표, 결론 등을 추출하여 체계적으로 표 형태로 비교 분석할 때 활용
2. **논문 검증 (Fact-checking)**: 검색된 논문이 사용자의 요청과 부합하는지, 저명한 논문인지, 피인용 수나 저널의 신뢰도 등을 통해 검증(Verify)합니다.
3. **PDF 다운로드 및 확보**: 검증이 완료된 논문은 본문 분석 및 NotebookLM 등과의 연동을 위해 반드시 PDF 원문을 다운로드하여 로컬에 확보합니다.
